## 技术要求

- Next.js 16 App Router + Turbopack
- React 19 + TypeScript 5 (路径别名 @/* → ./src/*)
- Tailwind CSS 4 + Radix UI 组件库
- Zustand 全局状态 + TanStack React Query 服务端数据
- RainbowKit/Wagmi 钱包连接 + ethers 6 链上交互
- Axios 请求 + bignumber.js 大数处理
- 包管理: Yarn

## 编码原则

1. 组件拆分到 Radix UI 可直接复用
2. 页面级加载/空/错误状态必须覆盖
3. 响应式至少 3 断点（375/768/1200）
4. API 层统一用 Axios + interceptor
5. 编码完成后运行 `yarn build` 验证

## 🛑 Red Flags 自检（被动技能: Superpowers/Discipline）

| 你在想什么 | 实际应该做什么 |
|-----------|-------------|
| "跳过 build 验证直接输出" | build 不可跳过 |
| "这个边界不会出现" | 写 loading/empty/error 状态，让证据说话 |
| "先核心功能再错误处理" | 错误处理是核心功能的一部分 |
| "我知道代码是对的" | 跑 `yarn build`，让编译器证明 |
| "顺便改个不相关的东西" | 不要。只做 PRD 范围内的事 |
| "差不多就行了" | 完整实现或不做 |

## ⚡ 完成前验证（被动技能: Superpowers/Verification-Before-Completion）

在输出 `"success": true` 之前，必须逐条确认：

- [ ] 所有页面覆盖了 loading / empty / error 三种状态
- [ ] `yarn build` 编译成功（实际运行了，不是"应该通过"）
- [ ] `yarn lint` 无新增 warning
- [ ] 所有 API 调用都有错误处理
- [ ] 没有跳过的 Phase
- [ ] 没有硬编码的 API URL / 密钥
- [ ] 响应式至少覆盖 375/768/1200

**任何一条未满足 → `success` 必须为 `false`。**
