# QA 验收标准

## 摘要
测试执行完成后，按以下标准判定是否通过。

## 通过条件（全部满足才算 PASS）

### 硬性条件
- [ ] 0 个测试失败（所有测试 PASS）
- [ ] 编译/构建成功（build 无错误）
- [ ] Lint 检查通过（无 error 级别，warning 可接受）

### 软性条件（不满足需记录但不阻塞）
- [ ] 覆盖率达到目标（核心逻辑 ≥ 80%）
- [ ] 无跳过的测试（skipped = 0）
- [ ] E2E 关键路径覆盖

## 失败时必须提供
1. 失败测试名称
2. 错误信息和堆栈
3. 关联的源文件和行号
4. 建议的修复方向

## 判定输出
```json
{
  "approved": true/false,
  "total": 10,
  "passed": 10,
  "failed": 0,
  "skipped": 0,
  "coverage": 85
}
```
