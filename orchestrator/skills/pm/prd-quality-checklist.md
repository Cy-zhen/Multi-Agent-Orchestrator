# PRD 质量检查清单

## 摘要
PRD 必须通过以下检查项才算合格。写完后自我检查。

## 检查项

### 必须有
- [ ] 一句话产品定义
- [ ] 目标用户画像（至少 1 个 Persona）
- [ ] 核心功能列表（每个用 IPO 格式）
- [ ] 非功能需求（P95 延迟、QPS、存储量级等具体数值）
- [ ] 可测试的验收标准（QA 能直接写 test case）
- [ ] 数据模型 / ER 图
- [ ] 复杂流程的 Mermaid 状态图

### 不允许
- [ ] 没有 "待定"、"TBD"、"后续补充" 等占位文字
- [ ] 没有模糊描述（"尽可能快"→ 改为 "P95 < 200ms"）
- [ ] 没有脱离技术栈的需求（如要求 CRE 但项目不用该框架）
