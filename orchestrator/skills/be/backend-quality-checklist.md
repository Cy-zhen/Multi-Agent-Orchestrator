# 后端质量清单

## 摘要
后端代码实现完成后，用此清单自查。

## 检查项

### 安全性
- [ ] 所有用户输入已校验（类型、长度、注入防护）
- [ ] 认证/授权中间件正确挂载
- [ ] 敏感数据不出现在日志和响应中
- [ ] SQL 查询使用参数化（防 SQL 注入）
- [ ] CORS 配置正确（不是 `*`）

### 性能
- [ ] 数据库查询有索引覆盖（EXPLAIN 验证）
- [ ] N+1 查询已优化（用 JOIN 或 batch）
- [ ] 大量数据用分页，不返回全量
- [ ] 耗时操作异步化（队列/goroutine）

### 可观测性
- [ ] 关键操作有结构化日志（JSON 格式）
- [ ] 请求链路有 request_id
- [ ] 错误有 stack trace 记录
- [ ] 健康检查端点 `/health` 存在

### 代码质量
- [ ] 无硬编码配置（环境变量或配置文件）
- [ ] 数据库操作在 Repository/DAO 层
- [ ] 业务逻辑在 Service 层
- [ ] 单元测试覆盖核心逻辑
