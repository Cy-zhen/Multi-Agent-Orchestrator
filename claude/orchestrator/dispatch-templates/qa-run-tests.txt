## 执行步骤

1. 读取测试计划 (`doc/test-plan.md`)
2. 根据实际代码结构编写测试脚本
3. 运行所有测试:
   - 后端: `go test ./...`
   - 前端: `yarn test` 或 `npx jest`
4. 收集测试结果（通过/失败/跳过）
5. 判定: 全部通过 → approved: true / 有失败 → approved: false

## 输出要求

- 失败测试必须包含: 测试名、错误描述、文件和行号
- 覆盖率数据（如可获取）
