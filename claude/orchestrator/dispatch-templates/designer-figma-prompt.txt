## 输出要求

### 提示词结构

每批提示词包含:
1. **Style Prefix**: 设计风格/主色调/字体/圆角
2. **Stitch Design Tokens**: Tailwind CSS config 格式的 color/font/spacing tokens
3. **Web Responsive Design Rules**: 响应式断点规范
4. **App Context**: 应用名/角色/强调色/具体美学方向
5. **Pages**: 每个页面 Mobile (375x812) + Desktop (1440x900)

### 批次拆分规则

- 每批提示词总长度 ≤ 5000 字符
- 按页面逻辑分组，每批 2-4 个页面
- 编号从 01 开始

### 每页必须包含

- 布局描述、组件列表、状态说明
- 默认态 / 加载态 / 空态 / 错误态
- mobile 和 desktop 需要独立描述，不是简单放大

### Stitch 交付物提醒

在提示词末尾提醒用户：完成 Stitch 设计后，需导出以下交付物给 FE Agent：
- **Code to Clipboard**: 完整 HTML 页面（含 Tailwind CSS config → colors/fontFamily/borderRadius）
- **.zip 导出**: 切图资源（icon/图片）
- **Stitch 分享链接**: 用于记录到 state.json

## 输出

保存到 `doc/stitch-prompts.md`（如有多批次，额外保存 `doc/stitch-batch-{NN}.md`）
