## 技术要求

- Go-zero 框架（handler → logic → svc 分层）
- GORM 定义模型并 AutoMigrate
- MySQL 数据存储，Redis 缓存热数据

## 编码原则

1. handler 只负责参数校验和响应
2. 业务逻辑全部在 logic 层
3. svc 负责依赖注入
4. Redis key 命名: `{module}:{entity}:{id}`
5. 性能敏感路径加 benchmark
6. 编码完成后运行 `go test ./...` 验证
